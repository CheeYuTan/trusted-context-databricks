Working title:
Modeling Business Semantics with Databricks Metric Views: LOD, Windows, Metadata, and Materialization

Core thesis:
Metric Views have moved beyond reusable aggregations. They can now model business semantics across calculation grains, expose AI-friendly metadata, express time intelligence, and accelerate dashboard queries through materialization.

Article structure:

1. Recap the previous blog
   - The previous article showed finance analytics with joins, nested measures, rolling metrics, YTD, and semiadditive balances.
   - The sequel focuses on what has improved: LOD expressions, richer window semantics, agent metadata, and materialization.

2. Why calculation grain matters
   - Finance users ask questions at different grains: transaction, month, quarter, entity, product family, account category.
   - Traditional SQL often duplicates denominator logic across dashboards.
   - Metric Views let the semantic layer own the calculation grain.

3. Build the tutorial data model
   - fact_gl_transactions: journal-line actuals.
   - fact_monthly_targets: monthly budget and forecast.
   - fact_month_end_balances: semiadditive balances.
   - Shared dimensions: account, product, entity, segment, calendar, scenario.
   - Include ERD and grain map diagrams.

4. Define the Metric View
   - Use YAML 1.1.
   - Define fields for fiscal and business hierarchies.
   - Define atomic measures first.
   - Use MEASURE() composability for gross profit, EBITDA, margin, variance, and growth.
   - Keep the base Metric View non-materialized so semantic modeling and performance optimization are taught separately.

5. Fixed LOD expressions
   - Show global revenue and account-category revenue as fixed LOD fields.
   - Show product-family revenue as a hierarchy-level fixed LOD field.
   - Use ANY_VALUE() when referencing fixed LOD fields in measures.
   - Explain that fixed LOD is calculated at a predefined grain.
   - Demonstrate query-time filter behavior by filtering to APJ while keeping the fixed global denominator.

6. Coarser LOD expressions
   - Use window measures with range: all.
   - Demonstrate percent of region revenue by entity.
   - Demonstrate excluding multiple fields with range: all for entity and product family.
   - Explain the difference from fixed LOD: coarser LOD can remain filter-aware.

7. Window semantics
   - Show current month revenue.
   - Show running total revenue.
   - Show YTD revenue using multiple window specs.
   - Show rolling 12-month revenue using trailing 12 month inclusive.
   - Compare trailing 3 month inclusive vs exclusive.
   - Show next-month revenue using leading 1 month.
   - Show prior-year revenue using offset: -12 month.
   - Compose YoY growth and YoY growth percentage.
   - Explain why hierarchy fields should be defined from the order field.

8. Semiadditive balances
   - Show why balances should not be summed across months.
   - Use range: current and semiadditive: last.
   - Demonstrate quarter-level query behavior.

9. Agent metadata
   - Add display_name, comment, format, and synonyms.
   - Show examples for revenue, EBITDA, margin, YTD, rolling 12, and month-end balance.
   - Explain why this matters for dashboards and Genie.

10. Materialization
   - Create a separate materialized Metric View variant for comparison.
   - Add an unaggregated materialization for expensive source preparation.
   - Add aggregated materializations for executive and drilldown dashboard shapes.
   - Explain exact match, rollup match, and unaggregated fallback.
   - Explain why additive measures can roll up and non-additive measures such as COUNT(DISTINCT) need exact match or fallback.
   - Use EXPLAIN EXTENDED to verify query rewrite.
   - Use DESCRIBE EXTENDED to inspect refresh information.
   - Mention serverless/runtime requirements and Public Preview status.

11. Dashboard showcase
   - Executive overview.
   - LOD drilldown.
   - Window semantics.
   - Semiadditive balances.
   - Materialization verification.
   - Genie-style prompts.

12. Conclusion
   - The old story was "define metrics once."
   - The new story is "model business semantics once, across grains, tools, and performance paths."
